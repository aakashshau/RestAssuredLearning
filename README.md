# Rest Assured API Automation Testing Project

This repository contains the code and test resources I used to practice and strengthen my skills in **API automation using Rest Assured** with **TestNG** and **Java**. The project is part of my hands-on learning journey in RESTful service testing and serves as a base framework for validating different API flows, including authentication, data serialization/deserialization, and complex request-response validations.

---

## 🔍 Project Overview

* **Framework Used:** Rest Assured (Java-based library for API testing)
* **Testing Framework:** TestNG
* **Build Tool:** Maven
* **IDE:** Eclipse
* **Reporting:** TestNG HTML Reports

---

## 🧠 Skills Practiced

### ✅ API Automation with Rest Assured

* Validated RESTful endpoints using various HTTP methods (GET, POST, DELETE).
* Tested complex payloads and dynamic JSONs.

### ✅ Request Specification and Response Specification

* Reused common headers and base URIs using builder patterns.

### ✅ OAuth 2.0 & E-Commerce API Flow

* Implemented token-based authentication.
* Automated login → add product → place order → validate order flow.

### ✅ Serialization and Deserialization

* Used POJOs to map JSON request bodies and responses for better code structure and readability.

### ✅ Dynamic JSON Parsing

* Parsed nested and dynamic JSON responses using `JsonPath`.

### ✅ Reusable Utilities

* Created utility methods for token generation, request execution, and response extraction.

### ✅ TestNG XML Configuration

* Defined test suites and parallel execution setup via `testng.xml`.

## 🧪 Key Test Cases Included

* `Basics.java`: Basic GET/POST tests using raw JSON and parameters
* `OAuth.java`: Authentication with access token usage
* `ECommerceAPITest.java`: Full workflow from login to placing and verifying orders
* `ComplexJsonParse.java`: Navigating nested JSON structures
* `SpecBuilderTest.java`: Centralized request and response specifications
* `serializeTest.java`: Serializing POJOs into request payloads
